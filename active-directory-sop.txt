================================================================================
STANDARD OPERATING PROCEDURE: AUTOMATED ACTIVE DIRECTORY & GROUP POLICY LAB
================================================================================

This Standard Operating Procedure (SOP) provides an end-to-end guide for initializing 
a local repository in VS Code and fully automating a Windows Server Active Directory 
Domain Services (AD DS) ecosystem along with a corporate Group Policy baseline.

--------------------------------------------------------------------------------
SECTION 1: LOCAL REPOSITORY SETUP (VS CODE TERMINAL)
--------------------------------------------------------------------------------
Before modifying the server environment, initialize a local Git repository to 
track your infrastructure configurations, scripts, and documentation.

1. Open VS Code and launch a new terminal (Ctrl + ~).
2. Create and navigate to your project directory:
   
   mkdir active-directory-lab
   cd active-directory-lab

   * mkdir: Short for "make directory". It creates a brand-new folder.
   * cd: Short for "change directory". Shifts your terminal focus into that folder.

3. Initialize the Git repository:
   
   git init

   * git init: Creates a hidden .git directory structure. This officially turns 
     the folder into a Git repository to track file modifications and branches.

4. Create a basic configuration and tracking structure:
   
   echo "# Active Directory Lab Portfolio" > README.md
   mkdir scripts

5. Stage and commit your baseline environment:
   
   git add .
   git commit -m "initial commit: project environment baseline setup"

   * git add .: Stages all newly created folders/files for a tracking snapshot.
   * git commit: Permanently records your staged changes into local Git history.

--------------------------------------------------------------------------------
SECTION 2: AUTOMATED LAB DEPLOYMENT (INSIDE WINDOWS SERVER VM)
--------------------------------------------------------------------------------
Log into your Windows Server VM via RDP, and open POWERSHELL AS AN ADMINISTRATOR.

PHASE 1: FEATURE & ROLE INSTALLATION
------------------------------------
Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools
Install-WindowsFeature -Name GPMC

* Install-WindowsFeature: Adds roles, role services, and features to Windows Server.
* -Name AD-Domain-Services: Installs Active Directory Domain Services, configuring 
  the core database engine that stores all user, group, and machine identities.
* -IncludeManagementTools: Ensures management tools (like ADUC) and the AD 
  PowerShell module are installed along with the server binaries.
* -Name GPMC: Installs the Group Policy Management Console.

Note: Close and reopen Server Manager to allow console menus to refresh.

PHASE 2: FOREST & DOMAIN PROMOTION
----------------------------------
Import-Module ADDSDeployment
$DSRMPassword = ConvertTo-SecureString 'YourStrongDSRMPassword!' -AsPlainText -Force

Install-ADDSForest `
    -DomainName 'lab.local' `
    -DomainNetBiosName 'LAB' `
    -InstallDns:$true `
    -SafeModeAdministratorPassword $DSRMPassword `
    -Force:$true

* Import-Module ADDSDeployment: Loads deployment cmdlets into the session.
* Install-ADDSForest: Initiates the creation of a brand-new Active Directory forest.
* -DomainName 'lab.local': Defines the primary root domain boundary.
* -DomainNetBiosName 'LAB': Flat-name identifier used for legacy authentication.
* -InstallDns:$true: Installs the DNS Server role (AD completely relies on DNS).
* ConvertTo-SecureString: Converts cleartext to an encrypted string for DSRM recovery.
* -Force:$true: Suppresses safety prompts so the system can automate and reboot.

WARNING: The server will automatically reboot upon successful completion.

PHASE 3: DIRECTORY ARCHITECTURE & ACCOUNT PROVISIONING
------------------------------------------------------
Log back in as LAB\Administrator, open a new PowerShell Admin window, and run:

# 1. Automated Organizational Unit (OU) Provisioning
New-ADOrganizationalUnit -Name "IT" -Path "DC=lab,DC=local"
New-ADOrganizationalUnit -Name "Finance" -Path "DC=lab,DC=local"
New-ADOrganizationalUnit -Name "HR" -Path "DC=lab,DC=local"
New-ADOrganizationalUnit -Name "Sales" -Path "DC=lab,DC=local"
New-ADOrganizationalUnit -Name "Computers" -Path "DC=lab,DC=local"

# 2. Automated Security Group Provisioning
New-ADGroup -Name "IT_Admins" -GroupScope Global -GroupCategory Security -Path "OU=IT,DC=lab,DC=local"
New-ADGroup -Name "Finance_Users" -GroupScope Global -GroupCategory Security -Path "OU=Finance,DC=lab,DC=local"
New-ADGroup -Name "HR_Users" -GroupScope Global -GroupCategory Security -Path "OU=HR,DC=lab,DC=local"
New-ADGroup -Name "Sales_Users" -GroupScope Global -GroupCategory Security -Path "OU=Sales,DC=lab,DC=local"

# 3. Batch User Provisioning & Password Seeding
$password = ConvertTo-SecureString "Welcome@2026!" -AsPlainText -Force

New-ADUser -Name "alice.chen" -GivenName "Alice" -Surname "Chen" -SamAccountName "alice.chen" -UserPrincipalName "alice.chen@lab.local" -Path "OU=IT,DC=lab,DC=local" -AccountPassword $password -Enabled $true
New-ADUser -Name "bob.patel" -GivenName "Bob" -Surname "Patel" -SamAccountName "bob.patel" -UserPrincipalName "bob.patel@lab.local" -Path "OU=Finance,DC=lab,DC=local" -AccountPassword $password -Enabled $true
New-ADUser -Name "carol.jones" -GivenName "Carol" -Surname "Jones" -SamAccountName "carol.jones" -UserPrincipalName "carol.jones@lab.local" -Path "OU=HR,DC=lab,DC=local" -AccountPassword $password -Enabled $true
New-ADUser -Name "david.smith" -GivenName "David" -Surname "Smith" -SamAccountName "david.smith" -UserPrincipalName "david.smith@lab.local" -Path "OU=Sales,DC=lab,DC=local" -AccountPassword $password -Enabled $true

# 4. Automated Role-Based Group Memberships
Add-ADGroupMember -Identity "IT_Admins" -Members "alice.chen"
Add-ADGroupMember -Identity "Finance_Users" -Members "bob.patel"
Add-ADGroupMember -Identity "HR_Users" -Members "carol.jones"
Add-ADGroupMember -Identity "Sales_Users" -Members "david.smith"

* New-ADOrganizationalUnit: Generates OUs to partition resources logically.
* -Path "DC=lab,DC=local": Uses a Distinguished Name (DN) to establish parent placement.
* New-ADGroup: Generates security group objects for Role-Based Access Control (RBAC).
* New-ADUser: Provisions human identities into the AD database.
* Add-ADGroupMember: Maps users into security groups to inherit group privileges.

--------------------------------------------------------------------------------
SECTION 3: GROUP POLICY AUTOMATION (STEP 5 DEPLOYMENT)
--------------------------------------------------------------------------------
Import-Module GroupPolicy

# Create and Link GPO
$GPO = New-GPO -Name "IT Security Policy" -Comment "Automated IT OU Security Baseline"
New-GPLink -Name "IT Security Policy" -Target "OU=IT,DC=lab,DC=local"

# Inject Inactivity Screensaver Timeout via Registry Settings
Set-GPRegistryValue -Name "IT Security Policy" -Key "HKCU\Software\Policies\Microsoft\Windows\Control Panel\Desktop" -ValueName "ScreenSaveActive" -Type String -Value "1"
Set-GPRegistryValue -Name "IT Security Policy" -Key "HKCU\Software\Policies\Microsoft\Windows\Control Panel\Desktop" -ValueName "ScreenSaverIsSecure" -Type String -Value "1"
Set-GPRegistryValue -Name "IT Security Policy" -Key "HKCU\Software\Policies\Microsoft\Windows\Control Panel\Desktop" -ValueName "ScreenSaveTimeOut" -Type String -Value "900"

* New-GPO: Spawns an empty Group Policy Object container template.
* New-GPLink: Maps the GPO to our targeted OU, forcing objects inside to inherit rules.
* Set-GPRegistryValue: Targets specific system registry paths to configure a 
  locked screensaver timeout after 900 seconds (15 minutes) of user inactivity.

--------------------------------------------------------------------------------
SECTION 4: AUTOMATED VERIFICATION AUDIT
--------------------------------------------------------------------------------
Run this block to quickly check all configurations:

Get-ADDomainController | Select-Object ComputerName, OperatingSystem, Forest
Get-ADOrganizationalUnit -Filter * | Select-Object Name
Get-ADUser -Filter {Enabled -eq $true} | Select-Object Name, SamAccountName
Get-ADGroupMember -Identity "IT_Admins" | Select-Object Name
Get-GPInheritance -Target 'OU=IT,DC=lab,DC=local' | Select-Object -ExpandProperty GpoLinks
